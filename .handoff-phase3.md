KEY DESIGN PRINCIPLES:
- aa2-ffi is a THIN translation layer — no game logic, just serialize/deserialize + call aa2-game
- All complex data crosses FFI as JSON strings (debuggable, Unity-friendly)
- Rust owns all game state via opaque GameContext pointer
- Unity copies strings immediately after receiving them
- Error handling: return {"error": "message"} JSON on failure
- Performance budget: tick <1ms, player_action <1ms, run_combat <50ms, get_player_view <5ms
